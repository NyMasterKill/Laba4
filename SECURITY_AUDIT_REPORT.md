# Отчет по безопасности приложения Laba4

## Аудит зависимостей
- Выполнено: 8 декабря 2025 года
- Уязвимостей не обнаружено
- Статус: Безопасно

## Защиты, реализованные в приложении

### 1. Защита от SQL-инъекций
- Используется TypeORM с параметризованными запросами
- Все пользовательские данные валидируются через Zod перед использованием в запросах

### 2. JWT-аутентификация и ротация токенов
- Access-токены действуют 15 минут
- Refresh-токены действуют 7 дней
- Refresh-токены хэшируются перед сохранением в БД
- Механизм отзыва токенов реализован через таблицу UserSession

### 3. Rate limiting
- Общий рейт-лимит: 100 запросов в минуту на IP-адрес
- Повышенный рейт-лимит для аутентификации: 5 попыток в 15 минут на IP-адрес

### 4. Защита от других атак
- Используется helmet.js для настройки заголовков безопасности (в будущем можно добавить)
- CORS-политики ограничены
- Валидация входных данных через Zod/middlewares

## Рекомендации по безопасности

1. Регулярный аудит зависимостей (npm audit)
2. Проверка на уязвимости с помощью инструментов SAST
3. Использование HTTPS в production
4. Настройка firewall для ограничения доступа к API
5. Ведение логов безопасности и аудит действий пользователей

## Заключение
На момент аудита 8 декабря 2025 года приложение соответствует базовым требованиям безопасности.
